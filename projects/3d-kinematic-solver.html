3D Kinematic Solver

Python Kinematics 3D Geometry Nov 2025

Overview

  • Purpose: Custom-built 3D kinematic solver developed in Python from scratch to analyze suspension behavior throughout its travel
  • Inputs: Chassis hardpoints, upright-relative hardpoints, and arm lengths define the geometric constraints
  • Outputs: Calculates upright position and orientation changes across stipulated wheel travel range
  • Application: Crucial tool for early-stage suspension design, enabling rapid iteration and kinematic optimization
  • Advantage: Purpose-built alternative to expensive commercial software (ADAMS, OptimumKinematics) with complete transparency into underlying algorithms
  • Versatility: Handles arbitrary suspension geometries—from simple double-wishbone to complex multi-link systems

Why Build a Custom Solver?

  • Rapid Iteration: Engineers need to quickly evaluate how geometric changes affect kinematic behavior during suspension development
  • Complete Transparency: Unlike commercial software, a custom solver provides full insight into the underlying computational methods and algorithms
  • Design Integration: Enables seamless integration into automated optimization routines and parametric design workflows
  • Deep Understanding: Building from scratch deepens comprehension of the fundamental geometric relationships governing suspension kinematics
  • Cost Efficiency: Eliminates expensive licensing fees while providing equivalent analytical capabilities
  • Extensibility: Provides a foundation for future enhancements like compliance analysis, kinematic synthesis, or custom metrics

The Challenge

Suspension kinematic analysis is fundamental to vehicle dynamics but presents several computational challenges:

  • 3D Spatial Geometry: Solving the position and orientation of the upright requires handling complex 3D coordinate transformations and constraint equations
  • Multi-Body Kinematics: Each suspension link constrains the upright's motion, creating a system of geometric constraints that must be solved simultaneously
  • Computational Efficiency: For design optimization, the solver must analyze multiple configurations quickly
  • Accuracy Requirements: Kinematic outputs directly affect vehicle handling characteristics, requiring sub-millimeter precision
  • Visualization: Understanding suspension behavior requires clear visualization of motion patterns and kinematic curves

The Solution

The solver was developed entirely in Python using fundamental principles of 3D geometry and kinematics:

  • Input System: Accepts chassis hardpoints (fixed points on the chassis), upright-relative hardpoints (points defined relative to the wheel center), and link lengths as geometric constraints
  • Constraint Solving: Implements 3D geometric algorithms to solve for upright position and orientation at each step of wheel travel, satisfying all link length and mounting point constraints
  • Kinematic Analysis: Calculates critical suspension parameters including camber gain, toe change (bump steer), scrub radius variation, and longitudinal wheel movement
  • Animation Output: Generates animated visualizations showing suspension geometry evolution through travel range
  • Performance Curves: Produces plots of kinematic characteristics versus wheel travel for design evaluation

Wheel Path Analysis

This plot shows how the tire contact patch moves laterally (scrub) and fore-aft (longitudinal) as the suspension compresses and extends. Minimizing scrub reduces tire wear and improves mechanical grip, while longitudinal movement affects anti-dive and anti-squat characteristics.

Camber and Toe Analysis

This is arguably the most critical plot for suspension design. Camber gain determines how the wheel tilts relative to the chassis during suspension compression, directly affecting tire contact patch and cornering performance. Toe change (bump steer) indicates unwanted steering input caused by suspension movement, which must be minimized for predictable handling and stability.

Technical Implementation

  • Language: Python 3.x
  • Core Libraries: NumPy for linear algebra operations, SciPy for numerical solving algorithms
  • Visualization: Matplotlib for static plots and animated GIF generation
  • Algorithm Approach: Iterative geometric constraint solving using spherical intersection methods to determine upright position, followed by orientation calculation from link attachment vectors
  • Performance: Full suspension travel analysis (50+ discrete positions) completes in under 1 second on standard hardware

Results and Impact

  • Design Tool: Successfully used for multiple race car suspension design projects, enabling rapid kinematic optimization
  • Speed Advantage: Reduced suspension analysis time from hours (CAD-based approach) to seconds, facilitating iterative design workflows
  • Accuracy: Achieved sub-millimeter precision in position calculations and sub-0.1° accuracy in orientation angles
  • Educational Value: Deepened understanding of 3D kinematics, constraint solving, and suspension design principles
  • Extensibility: Modular architecture allows easy addition of new kinematic outputs (roll center height, anti-geometry, etc.)

Key Learnings

  • Computational Geometry Fundamentals: Understanding spherical intersections, coordinate transformations, and constraint solving is essential for accurate kinematic analysis
  • Tool Development: Building specialized tools from scratch often provides better insight and control than using black-box commercial software
  • Validation Strategy: Cross-referencing results against multiple sources (analytical solutions, CAD measurements, professional software) builds confidence
  • Visualization Importance: Animated visualizations are crucial for understanding complex 3D motion and identifying geometric issues
  • Performance Optimization: Efficient algorithms enable rapid iteration, which is critical for design optimization workflows
